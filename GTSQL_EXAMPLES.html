GTSQL Examples Corpus

Paired gt (R) and GTSQL examples reproducing the same display table. Examples are drawn from Creating Effective Display Tables with the gt Package and the gt reference.

Only examples reproducible with the clauses defined in GTSQL_PLAN.md are included. Excluded gt features (no GTSQL equivalent today) include: tab_row_group() explicit row groups (basic groupname_col grouping is supported via FACET <column>), multi-column stubs, tab_stub_indent(), tab_footnote(), per-cell footnotes, row_order(), tab_spanner_delim(), cols_add(), grand_summary_rows(), text_transform() with arbitrary R functions, from_column() dynamic parameters, nanoplots, and tab_options() theme tuning.

Each example assumes a SQL data source whose schema matches the corresponding R dataset (e.g. a gtcars table mirroring the gtcars tibble).

GTSQL

WITH monthly AS (
  SELECT CAST(SUBSTR(date, 6, 2) AS INT) AS month, COUNT(*) AS pizzas_sold
  FROM pizzaplace
  GROUP BY 1
),
totals AS (SELECT SUM(pizzas_sold) AS total FROM monthly)
SELECT m.month, m.pizzas_sold, 1.0 * m.pizzas_sold / t.total AS pct_of_annual
FROM monthly m CROSS JOIN totals t

TABULATE month, pizzas_sold, pct_of_annual
FORMAT STUB month
FORMAT pizzas_sold
  RENAMING * => '{:num ,d}'
FORMAT pct_of_annual
  RENAMING * => '{:num .1f}%'

Notes on omissions

The following table types from the source book are intentionally not in the corpus because GTSQL cannot currently express them:

  • Multi-column stubs (rowname_col = c("region", "category")).
  • Tables with cell footnotes (tab_footnote()) or per-element styling of the table footer beyond caption.
  • Tables that re-order rows via row_order() / row_group_order() after creation — equivalent ordering must be done in the upstream SELECT ... ORDER BY.
  • Tables using tab_spanner_delim() to derive spanners from delimited column names — equivalent must be expressed via explicit FORMAT SPAN clauses.
  • Stub indentation (tab_stub_indent()).
  • Nanoplots (cols_nanoplot()).
  • grand_summary_rows() (table-wide totals; the closest GTSQL equivalent is a FACET clause with no grouping column, but this maps to summary_rows() semantics rather than a true grand summary).
  • text_transform() with arbitrary R functions; only literal value-to-string mapping is supported via RENAMING <value> => '<string>'.
  • from_column() dynamic parameters (e.g. per-row currency code or decimal precision pulled from another column).
  • Global theming via tab_options() / opt_*() helpers.
